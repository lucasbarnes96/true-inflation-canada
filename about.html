Transparent Inflation Nowcast Built on Public Data

True Inflation Canada is an experimental open-source project that estimates inflation pressure between official CPI releases. The goal is to provide transparent, auditable signal tracking using public data sources, while clearly documenting caveats and uncertainty.

Project Goal, Strengths, and Long-Term Vision

Primary Goal

Build a public, reproducible inflation nowcast that is useful before official release dates, without requiring paid data feeds. The project prioritizes transparency, auditability, and explicit quality gates over cosmetic precision.

Core Strengths

  • Open-source ingestion and methodology; assumptions are inspectable.
  • Source-level diagnostics, freshness checks, and publish gates.
  • Robust fallback behavior to avoid silent failures.
  • Explicit confidence and quality scoring tied to measurable inputs.

Known Shortcomings

  • Early live-history period is volatile and less statistically stable.
  • Some CPI components are omitted in v1 due to high-frequency data quality constraints.
  • Public source outages, anti-bot behavior, or certificate issues can temporarily degrade signal freshness.
  • This is not an official CPI publication and should not be treated as a replacement.

Long-Term Vision

  • Expand source diversity and reduce single-source category dependence.
  • Increase tracked basket coverage and improve representativeness over time.
  • Improve calibration quality as live history accumulates.
  • Maintain an open, finance-grade public benchmark for high-frequency inflation tracking.
How the Math Works

Core Formulas

  • category_contribution = weight x daily_change_pct
  • nowcast_mom = weighted aggregate of category deltas
  • projected_index = prior_index x (1 + mom_signal x prorate_factor)
  • nowcast_yoy = (projected_index / prior_year_index - 1) x 100

Signal Lifecycle vs CPI

  • Stage 1: Official CPI history provides anchor context and denominator reference.
  • Stage 2: Daily/weekly/monthly public inputs populate category proxies.
  • Stage 3: As live days accumulate, nowcast series lengthens and error metrics stabilize.
  • Stage 4: Deviation and forecast outputs are published only when guardrails pass.
  • Stage 5: Relative usefulness improves over time as non-seeded live history grows.

Methodology & Governance Notes

Policy and QA

    Transparency Commitments

    • All published runs include source diagnostics and release gate status.
    • Confidence and quality are tied to explicit coverage/diversity/anomaly criteria.
    • Fallback usage is surfaced in notes rather than hidden.
    • Method limits are disclosed in the dashboard and this page.
